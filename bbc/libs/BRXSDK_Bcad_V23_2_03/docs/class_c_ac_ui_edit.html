◆ DoAdUiMessage()

UX_EXPORT_FUNCTION ADUI_REPLY CAcUiEdit::DoAdUiMessage ( ADUI_NOTIFY  code,
UINT  ctrlID,
LPARAM  lPar 
)
protectedvirtual

Default handler for AdUi registered messages; can be overridden to perform special processing by applications; default handler processes kAdUiNotify_RestoreFocus and kAdUiNotify_Validate events, others are processed by CAdUiEdit handler.

Parameters
[in]codeidentifies the message/event to be processed
[in]ctrlIDthe ID of the control which sent/triggered the message (or NULL)
[in]lParadditional, message-depending parameter
Returns
ADUI_REPLY one of the status code; usual values are : kAdUiReply_Ok - message/event was correctly processed kAdUiReply_Nothing - message/event was not processed (ignored) kAdUiReply_Valid - edit control's content is valid kAdUiReply_NotValid - edit control's content is not valid

Reimplemented from CAdUiEdit.

◆ OnKillFocus()

UX_EXPORT_FUNCTION void CAcUiEdit::OnKillFocus ( CWnd *  pNewWnd)
protected

This is the default handler for WM_KILLFOCUS message, can be overridden.

Note
this default handler calls the base class implementation, then inspects the edit style of this control : if AC_ES_VAL_ONKILLFOCUS is set, then the handler will call the convert and validation functions; if AC_ES_CONV_ONKILLFOCUS is set, then only the convert function is called

UX_EXPORT_FUNCTION void CAcUiEdit::SetStyleMask ( DWORD  value)
virtual

This function specifies the edit and validation style flags, and checks for conflicts.

Parameters
[in]valuethe application-specific flags, combines with AC_ES_xxx flags
Note
this function checks for conflicting text and numeric style flags, and will shows an Alert in that case; bits 0-2 (0x0...0xFFF} are reserved for CAcUi ! only use bit 3...31 !

Reimplemented from CAdUiEdit.
