◆ getCustomPropertyValue()

UX_EXPORT_FUNCTION bool BrxComponentDefinition::getCustomPropertyValue ( const ACHAR *  categoryName,
const ACHAR *  propertyName,
AcValue &  value 
) const
virtual

This method finds a value of the existing custom property with @categoryName and @propertyName.

If the property exists the value stored for this component will be returned. NOTE: It is assumed that a custom property value can have the following types: Integer, Real, String, Boolean. But AcValue doesn't support the Boolean type explicitly. A boolean value can be represented by @value in two ways. The first is resbuf and boolean value is defined by its restype (RTT- true, RTNIL - false). The second is special strings for boolean values ("<bool::true>" - true, "<bool::false>" - false). @params[in] categoryName - the name of the property set which contains the considered property propertyName - the name of the given property itself @params[out] value - contains the property value set for this component

Returns
true if a value of the property was found successfully, the value of the particular type can be got from @value; false if the property was not found or its value was not set.

)
virtual

This method sets a value for the existing custom property with @categoryName and @propertyName.

NOTE: It is assumed that a custom property value can have the following types: Integer, Real, String, Boolean. But AcValue doesn't support the Boolean type explicitly. If you want to set a boolean value, you have to pass AcValue object filled in two ways. The first way is to set resbuf for the created value with the special restype: RTT is for true, RTNIL is for false. The second is to set the special strings related to boolean values ("<bool::true>" is for true, "<bool::false>" is for false). @params[in] categoryName - name of the property set which contains the considered property propertyName - name of the the property value - value to be set params - special flag which defines behavior for the case if the property doesn't exist

Returns
Acad::eOk if the value has been set successfully
